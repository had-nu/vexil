# Code of Conduct

We are professionals building security tooling. Our conduct guidelines are straightforward and non-negotiable.

## 1. Professionalism
Treat all contributors with respect. Debate ideas, not individuals. Provide constructive feedback during code reviews.

## 2. Focus
Keep discussions focused on technical merit, architecture, and security. Avoid political, ideological, or extraneous commentary in issues, PRs, and commit messages.

## 3. Direct Communication
Be clear, concise, and direct. We value efficient communication over verbose pleasantries. Say what needs to be said respectfully and move forward. 

## Enforcement
Disruptive behavior, harassment, or persistent failure to adhere to these straightforward professional standards will result in removal from the project.

Report issues directly to the repository owner.
